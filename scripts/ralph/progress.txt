## Codebase Patterns
- Typecheck: `uv run pyright src/` (pyright added as dev dep)
- Entry point: `uv run skillmanager` maps to `skillmanager.main:run`
- NiceGUI: run with `host="127.0.0.1"` for localhost-only
- Build system: hatchling, packages = ["src/skillmanager"]
- NiceGUI page: use `@ui.page("/")` inside `run()` for proper per-client isolation
- NiceGUI splitter: `ui.splitter(value=N)` where N is 0-100 (percentage), use `.before`/`.after` slots
- NiceGUI element click: use `element.on("click", lambda _e, x=x: handler(x))` to capture loop vars; add `# type: ignore[misc]` for pyright lambda type complaints
- NiceGUI state: use `dict[str, T | None]` containers for mutable state shared across closures (avoids unbound-var issues with pyright)
- NiceGUI dialog: `with ui.dialog() as dialog, ui.card():` — create inside a function and call `dialog.open()` at the end; creates a fresh dialog each time
- NiceGUI visibility toggle: set `element.visible = False` after the `with` block to hide initially; update in event handlers
- NiceGUI event handler types: use `Any` for event parameter type in handlers registered via `.on_value_change()` to satisfy pyright
- NiceGUI async handlers: define handler as `async def`, use `asyncio.to_thread(fn, *args)` for blocking calls (e.g. subprocess); NiceGUI runs async handlers as coroutines automatically
- NiceGUI dynamic sidebar rows: keep `dict[str, ui.column | None]` ref to a container; use `with container:` at any time to append new rows; elements added outside initial `with` block still render correctly
- git clone via subprocess: `subprocess.run(["git", "clone", url, str(dest)], capture_output=True, text=True)` — check `returncode == 0`; stderr has the error message on failure
