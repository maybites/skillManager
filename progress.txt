## Codebase Patterns
- `Project.skills_dir` is a `@property` returning `Path(self.path) / ".claude" / "skills"`
- `add_project(path)` in `operations.py` validates path + auto-creates `.claude/skills/` dir; returns `OperationResult`
- Container ref pattern: `{name}_container_ref: dict[str, ui.column | None]` initialized to `{"col": None}`, then assigned in layout; use `# type: ignore[arg-type,union-attr]` on the `with` statement
- `detect_skills(path)` returns `list[tuple[str, bool]]` — rel_path and enabled_by_default; folders in `_UNCHECKED_BY_DEFAULT` frozenset default to False
- `_render_source_detail(panel, source)` pattern: function takes the panel and calls `panel.clear()` internally, allowing it to re-render in-place (used for confirm callback too)
- When calling a render function that manages its own `panel.clear() + with panel:`, do NOT wrap the call in another `with panel:` from the caller
- Use `monkeypatch` to override `_CONFIG_FILE` and `CONFIG_DIR` in config module for tests
- TOML serialization via `tomllib` (stdlib, read) + `tomli-w` (write); convert dataclasses to plain dicts first
- Enum values stored as strings in TOML (e.g. `kind = "remote"`)
- Matrix cell state derived from filesystem: `os.path.exists()` for symlinks, `os.path.islink()` to distinguish
- NiceGUI closures over loop variables: use default arg trick (`cb=cb` in lambdas)
- `find_owning_source` uses `TYPE_CHECKING` guard for Source import
- Conflict indicator computed from all confirmed sources before rendering — build `set[str]` of names with count > 1
- Quality check is `pyright` (`.venv/bin/pyright`), not mypy
- `is_copy(path)` = `path.exists() and path.is_dir() and not path.is_symlink()`
